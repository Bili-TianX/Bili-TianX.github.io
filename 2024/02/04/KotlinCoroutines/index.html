KotlinCoroutines

发表于 2024-02-04

Coroutine

特点：

  • suspendable
  • run concurrently
  • now bound to particular thread
  • light weight

Structured Concurrency

内容：Coroutine只能运行在CoroutineScope中

意义：

  • 限定生命周期
  • 避免Coroutine丢失、泄露
  • 正确处理异常

CoroutineScope

用途：承载Coroutine

特点：所有Coroutine结束后才退出

CoroutineBuilder

用途：提供CoroutineScope

实例：

  • runBlocking（阻塞线程）
  • launch

Suspend function

特点：

  • 可在Coroutine中被调用
  • 可在函数内调用其他suspend function
  • 自定义coroutineScope：使用coroutineScope { ... }

Suspend vs Block

Job

内容：代表Coroutine的Handle

操作：

  • job.join() 显式等待Coroutine完成

Light Weight

  • thread：占用大量内存
  • coroutine：占用极少内存

一些函数

  • delay(ms) 挂起Coroutine ms 毫秒

  • repeat(x) { ... } 重复执行 x 次代码块

async

作用：创建返回特定结果的Coroutine

返回类型：Deferred<T>

操作：

  • await()
  • start()
  • Iterable<Deferred<T>>.awaitAll()

GlobalScope.launch

不推荐使用

作用：创建全局范围下的Coroutine（不受任何CoroutineScope限制）

launch(Dispatcher)

改变Coroutine运行的线程

  • Dispatchers.Default 线程池
  • Dispatchers.Main 主线程

withContext

修改Coroutine所在的线程

默认的context继承父亲

async 继承 Job

Job.cancel()

Job.setupCancellation()

Channel

  • send()

  • receive()

  • close()

  • Channel<T>(容量)

    • <空> 无缓冲

    • Int 指定的缓冲容量

    • UNLIMITED 缓存全部

    • CONFLATED

      Requests a conflated channel in the Channel(…) factory function. This is a shortcut to creating a channel with onBufferOverflow = DROP_OLDEST.

    • RENDEZVOUS

    • BUFFERED

Flow

flow 特点：可在其中用suspend函数，但其本身不被标记为suspend

flow仅在collect时才会执行

flow可用withTimeout()取消

flow可用.map { ... }、.filter { ... }等函数，且其中可用suspend函数

flow进行链式操作时，元素是一个个经过链式操作的，而非flow结束后才执行

错误用法（会抛出异常）：

正确用法：

flow { ... }.flowOn(调度器)

f().collectLatest { ... } 一旦有新的就重新开始

  • f().zip(f()) 按顺序

  • f().combine(f()) 按快慢

flow展平：

  • flatMapConcat
  • flatMapMerge
  • flatMapLatest
